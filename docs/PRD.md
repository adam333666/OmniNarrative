# **产品需求文档（PRD）终稿**

  

## **项目名称**

  

**面向内容创作者的零门槛多模态叙事内容生成平台**

---

# **1. 项目概述**

  

## **1.1 项目背景**

  

当前多数内容生成工具只解决单点问题，例如生成文案、生成脚本、生成图像提示词，或简单拼接多个模型能力。对内容创作者而言，真正的难点不是生成某一段内容，而是从一个模糊创意出发，快速得到一套：

- 内部表达自洽
    
- 面向目标平台可传播
    
- 面向目标受众有针对性
    
- 可直接进入后续多模态视频生成链路
    

  

的完整内容方案。

  

本项目目标不是一期直接生成成品视频，而是先构建一个**展示优先、前后端分离、结构清晰的系统原型**，输出一份**结构化、多层级、可解释、可导出、可供下游多模态系统消费的结果包**。

  

## **1.2 核心目标**

  

构建一个面向内容创作者的工作台，支持用户从一句创意意图出发，通过结构化引导，生成一套完整的多模态叙事内容方案，并显式体现平台、趋势、受众对内容设计的影响。

  

## **1.3 项目价值**

  

本项目解决的核心问题包括：

- 创作者不会把一个模糊想法扩展成完整方案
    
- 创作者不擅长为不同平台做表达适配
    
- 创作者缺少脚本、分镜、字幕、配音、封面、发布文案的一体化组织能力
    
- 现有生成工具结果分散，难以直接交给下游视频生成平台或制作流程
    

  

本项目的价值在于建立一条从创意输入到结构化内容输出的闭环链路。

  

## **1.4 目标用户**

  

当前阶段的目标用户为：

- 有明确创作意图的内容创作者
    
- 展示、答辩、演示场景中的体验用户
    
- 希望快速得到完整创作方案的用户
    

  

## **1.5 核心使用场景**

  

用户输入一句类似以下的话：

- 我想做一个关于时间旅行悖论的内容
    
- 我想做一个关于高三学生焦虑的内容
    
- 我想做一个讲猫为什么会踩奶的内容
    

  

系统通过结构化问答补充关键约束，输出一份可展示、可解释、可导出、可供下游消费的多模态叙事内容结果包。

# **3. 用户流程 / 页面流程**

  

## **3.1 关键角色**

  

### **创作者用户**

  

输入创意与约束，查看并导出结果。

  

### **系统管理员 / 开发者**

  

维护平台配置、趋势模板，手动触发趋势分析器，查看趋势分析结果。

  

## **3.2 页面结构**

  

系统一期固定为四页：

1. 首页
    
2. 输入向导页
    
3. 生成过渡页
    
4. 结果页
    

  

## **3.3 核心流程**

1. 用户进入首页
    
2. 点击开始创作
    
3. 进入五步向导输入页
    
4. 按顺序完成五个问题
    
5. 提交后进入生成过渡页
    
6. 系统分阶段执行生成链路
    
7. 用户进入结果页查看结果
    
8. 用户导出 Markdown / JSON，或获取 video payload
    

  

## **3.4 输入流程**

  

### **Step 1：主题 / 核心想法**

- 控件：多行文本框
    
- 形式：自由输入
    
- 必填
    
- 必须提供示例提示
    

  

### **Step 2：内容类型**

- 控件：单选卡片
    
- 选项：
    
    - 科普型
        
    - 故事型
        
    - 混合型
        
    - 自动判断
        
    
- 必填
    

  

### **Step 3：目标平台**

- 控件：单选卡片
    
- 选项：
    
    - 抖音
        
    - 快手
        
    - 小红书
        
    - B站
        
    - 视频号
        
    
- 必填
    

  

### **Step 4：目标受众**

- 控件：文本输入
    
- 形式：自然语言描述
    
- 必填
    
- 必须提供示例提示
    
- 后端负责抽取结构化标签
    

  

### **Step 5：风格 / 情绪基调**

- 控件：单选卡片 + 可选自定义补充
    
- 必填
    

  

## **3.5 输入流程约束**

- 流程为**不可回退式**
    
- 当前步未通过校验不可进入下一步
    
- 生成后不允许修改参数重生成
    
- 用户如需重做，只能重新开始一次新流程
    

  

## **3.6 生成状态**

  

生成过渡页必须展示以下状态：

1. 正在解析创作主题
    
2. 正在抽取受众与风格标签
    
3. 正在生成叙事骨架
    
4. 正在进行平台与趋势适配
    
5. 正在组装多模态内容包
    

  

## **3.7 结果页展示逻辑**

  

### **默认展开**

- 项目总览层
    
- 叙事脚本层
    
- 平台适配层
    

  

### **默认折叠**

- 多模态生成层
    
- 机器可执行提示层
    

  

### **左侧区域**

- 输入摘要
    
- 受众标签抽取结果
    
- 平台趋势摘要
    
- 关键设计决策说明
    
- 导出按钮区
    

  

### **右侧区域**

- 五层结果包内容

# **4. 功能需求**

  

## **4.1 首页模块**

  

### **目标**

  

提供产品说明和创作入口。

  

### **功能**

- 展示平台定位与价值
    
- 展示核心亮点：
    
    - 自洽创作
        
    - 平台适配
        
    - 多模态结果包
        
    
- 提供开始创作按钮
    

---

## **4.2 五步向导输入模块**

  

### **目标**

  

收集生成内容所需的核心约束。

  

### **输入**

  

五类输入字段。

  

### **输出**

  

统一结构化输入对象。

  

### **核心逻辑**

- 分步骤展示问题
    
- 每步展示示例提示
    
- 校验当前输入
    
- 将输入转为统一格式
    

  

### **边界情况**

- 用户主题过短
    
- 受众描述过泛
    
- 风格为空
    
- 平台未选
    

  

### **异常情况**

- 输入为空
    
- 超长
    
- 枚举值非法
    

---

## **4.3 输入归一化与校验模块（Input Orchestrator）**

  

### **目标**

  

将前端输入转换为规范化请求对象。

  

### **输入**

  

五步输入原始字段。

  

### **输出**

  

CreationRequest

  

### **核心逻辑**

- 去除冗余空白
    
- 校验枚举值
    
- 补齐默认值
    
- 统一字段结构
    

  

### **异常情况**

- 缺失必填项
    
- 不支持的平台值
    
- 风格非法
    
- 字段超限
    

---

## **4.4 受众与风格解析模块（Audience/Profile Parser）**

  

### **目标**

  

把自然语言受众描述和风格描述解析为结构化标签。

  

### **输入**

- 目标受众文本
    
- 风格文本
    
- 主题文本
    

  

### **输出**

- AudienceProfile
    
- StyleProfile
    

  

### **核心逻辑**

- 调用模型适配层做标签抽取
    
- 输出结构化标签
    
- 抽取失败时回退为宽泛标签
    

  

### **边界情况**

- 用户输入非常短
    
- 受众描述抽象
    
- 风格描述冲突
    

  

### **异常情况**

- 模型返回格式不完整
    
- 标签字段缺失
    

---

## **4.5 趋势策略服务（Trend Strategy Service）**

  

### **目标**

  

为主生成链路提供平台趋势摘要和策略模板。

  

### **输入**

- 目标平台
    
- 内容类型
    
- 受众标签（可选参与）
    

  

### **输出**

  

平台策略对象，包括：

- 平台表达偏好
    
- 开头钩子模式
    
- 节奏结构建议
    
- 标题 / 封面风格
    
- 受众偏好摘要
    
- 不建议表达方式
    
- 热点叙事倾向摘要
    

  

### **核心逻辑**

- 从数据库读取最新模板
    
- 模板缺失时读取默认平台模板
    
- 将策略对象返回给生成链路
    

  

### **异常情况**

- 数据库读取失败
    
- 模板字段不全
    
- 平台无模板
    

---

## **4.6 叙事生成模块（Narrative Generator）**

  

### **目标**

  

根据输入约束和平台策略生成叙事骨架与主体脚本。

  

### **输入**

- 主题
    
- 内容类型
    
- 平台
    
- 受众标签
    
- 风格标签
    
- 趋势策略模板
    

  

### **输出**

- 一句话梗概
    
- 主标题
    
- 分段脚本
    
- 关键分镜细化
    
- 备选标题 / 钩子
    

  

### **核心逻辑**

- 先生成叙事总纲
    
- 再生成分段脚本
    
- 对关键段落细化镜头
    
- 输出少量高价值备选建议
    

  

### **输出规范**

  

采用**分段脚本为主 + 关键分镜细化**的混合结构。

  

每段必须至少包含：

- 段落编号
    
- 段落标题
    
- 段落目标
    
- 旁白 / 对白
    
- 字幕文案
    
- 画面描述
    
- 情绪说明
    
- 节奏说明
    

  

关键段落额外包含：

- 镜头拆分
    
- 镜头时长建议
    
- 镜头重点
    
- 转场提示
    

  

### **异常情况**

- 分段过少或过多
    
- 关键字段缺失
    
- 结构不完整
    

---

## **4.7 结果包组装模块（Package Assembler）**

  

### **目标**

  

将生成结果扩展并组装为完整五层结果包。

  

### **输入**

- 叙事脚本
    
- 平台策略
    
- 受众标签
    
- 风格标签
    

  

### **输出**

  

NarrativePackage

  

### **五层结构**

  

#### **第 1 层：项目总览层**

- 主标题
    
- 一句话梗概
    
- 内容定位
    
- 目标平台
    
- 目标受众摘要
    
- 风格摘要
    
- 设计摘要
    

  

#### **第 2 层：叙事脚本层**

- 分段脚本
    
- 关键分镜细化
    
- 脚本说明
    
- 标题 / 钩子备选
    

  

#### **第 3 层：多模态生成层**

- 角色设定
    
- 场景设定
    
- 视觉风格说明
    
- 字幕风格建议
    
- 音效 / BGM 建议
    
- 画面关键词
    
- 节奏建议
    

  

#### **第 4 层：平台适配层**

- 平台适配策略
    
- 趋势摘要
    
- 受众适配说明
    
- 钩子设计原因
    
- 节奏结构原因
    
- 标题 / 封面风格建议
    
- 发布文案建议
    
- 不建议表达方式
    
- 封面文案备选
    

  

#### **第 5 层：机器可执行提示层**

- 视频生成提示块
    
- 角色一致性描述
    
- 场景描述块
    
- 风格约束
    
- 负面约束
    
- 关键镜头 prompt
    
- 可选镜头时长建议
    

  

### **异常情况**

- 某层字段缺失
    
- Schema 校验失败
    
- 层间内容冲突
    

---

## **4.8 中间分析展示模块**

  

### **目标**

  

向用户展示系统具有分析与设计能力，而不是黑箱生成。

  

### **必须展示**

- 受众标签抽取结果
    
- 平台趋势摘要
    
- 关键设计决策说明
    

  

### **作用**

- 提升展示说服力
    
- 强化“外部设计引擎”存在感
    
- 增强可解释性
    

---

## **4.9 导出模块（Export / Payload Service）**

  

### **目标**

  

将统一结果结构输出为展示用和机器用格式。

  

### **输入**

  

统一结果包 JSON。

  

### **输出**

- Markdown
    
- JSON
    
- Video payload
    

  

### **设计原则**

- JSON 是系统唯一源数据格式
    
- Markdown 由模板从 JSON 渲染生成
    
- Video payload 从已有结果映射生成
    
- 不允许 Markdown 反向作为系统真值
    

  

### **异常情况**

- Markdown 渲染失败
    
- JSON 序列化失败
    
- Payload 校验失败
    

---

## **4.10 Video payload 接口模块**

  

### **目标**

  

为后续接入多模态视频生成系统预留标准化接口。

  

### **输入**

  

统一结果包。

  

### **输出**

  

VideoGenerationPayload

  

### **一期输出粒度**

- 视频元信息
    
- 分段脚本数组
    
- 关键镜头数组
    
- 角色 / 场景设定
    
- 风格约束
    
- 字幕文本块
    
- 音效 / 节奏建议
    
- 负面约束
    

  

### **约束**

- 一期不直接生成成品视频
    
- 不强绑定某一具体视频模型供应商
    
- 输出应保持通用性和扩展性
    

---

## **4.11 趋势抓取分析器**

  

### **目标**

  

通过手动触发方式抓取平台样本内容并生成趋势摘要与策略模板。

  

### **输入**

  

目标平台公开内容样本。

  

### **输出**

- 趋势摘要
    
- 平台策略模板
    

  

### **一期主抓平台**

- 抖音
    
- 小红书
    
- B站
    

  

### **一期辅抓 / 简化平台**

- 快手
    
- 视频号
    

  

### **触发方式**

- 命令行触发
    
- 内部 API 触发
    

  

### **一期不包含**

- 定时自动调度
    
- 独立后台页面

# **5. 技术方案要求**

  

## **5.1 前端技术栈**

- **Next.js**
    
- React 组件体系
    
- 向导式工作台结构
    
- 分阶段状态展示
    
- 结果分层展示
    

  

## **5.2 后端技术栈**

- **FastAPI**
    
- Python
    
- 负责生成编排、模板读取、导出和 payload 输出
    

  

## **5.3 架构**

- **前后端分离**
    
- 前端负责交互和展示
    
- 后端负责生成链路和服务编排
    

  

## **5.4 数据库**

- **PostgreSQL**
    
- 存储：
    
    - 趋势摘要
        
    - 平台策略模板
        
    - 平台配置
        
    - 系统预设
        
    
- 不存用户历史结果
    

  

## **5.5 缓存**

- 一期不强依赖 Redis
    
- 不将缓存作为核心依赖
    
- 后续可扩展模板读取缓存
    

  

## **5.6 队列 / 异步任务**

- 一期不引入独立任务队列
    
- 使用后端分阶段处理 + 前端轮询实现状态更新
    

  

## **5.7 模型接入方式**

- 建立**轻量模型适配层**
    
- 不直接耦合单一厂商
    
- 可通过配置替换底层模型
    

  

## **5.8 内部结果结构原则**

- 系统内部以统一 JSON Schema 为真值
    
- 页面展示、Markdown 导出、JSON 导出、Video payload 都必须从同一源结构派生
    

  

## **5.9 API 风格**

- REST 为主
    
- 核心接口少而清晰
    
- 内部趋势接口与用户接口分开
    

  

## **5.10 配置管理**

  

### **数据库存储**

- 趋势模板
    
- 平台配置
    
- 趋势摘要
    

  

### **配置文件 / 环境变量**

- 固定枚举
    
- 示例文案
    
- 模型配置
    
- 数据库连接
    
- 内部 API 密钥
    

  

## **5.11 日志与监控**

  

一期至少记录：

- 输入摘要
    
- 各生成阶段开始/结束
    
- 模板读取结果
    
- 模型调用耗时
    
- 导出状态
    
- 错误阶段与原因
    

  

## **5.12 错误处理**

  

必须支持：

- 输入校验失败：前端阻断
    
- 某生成阶段失败：明确提示失败阶段
    
- 趋势模板读取失败：回退默认模板
    
- 导出失败：保留结果页并提示
    
- 超时：返回明确提示，不保留任务
    

  

## **5.13 重试机制**

- 不做复杂自动重试
    
- 内部模型调用可有限重试
    
- 模板读取失败走默认回退
    

  

## **5.14 幂等性要求**

- 同一结果导出 JSON / Markdown 结构应一致
    
- Video payload 必须与当前结果页内容一致
    
- 趋势刷新接口不得重复写入脏数据
    

  

## **5.15 安全要求**

- 内部趋势 API 做最小权限保护
    
- 不长期保存用户创作结果
    
- 输入做基本长度和格式校验
    
- 导出接口避免任意路径风险
    

  

## **5.16 部署方式**

- **Docker 容器化**
    
- 一期拆分 3 个服务：
    
    - frontend
        
    - backend
        
    - db
        
    

  

## **5.17 环境隔离**

  

至少区分：

- 本地开发环境
    
- 演示 / 测试环境
    

  

## **5.18 测试要求**

  

至少覆盖：

- 输入校验测试
    
- JSON Schema 输出测试
    
- 趋势模板读取测试

# **9. 后端模块拆分**

    负责输入接收、校验、归一化

# **11. 非功能需求**

  

## **11.1 性能**

- 展示型系统允许一定生成耗时
    
- 必须有阶段反馈
    
- 不要求极低延迟，但不能长时间无响应
    

  

## **11.2 稳定性**

- 模板读取失败必须回退
    
- 导出失败不影响结果页展示
    
- 任一失败必须有明确阶段提示
    

  

## **11.3 可扩展性**

- 模型供应商可切换
    
- 可扩展接入视频生成系统
    
- 可扩展历史记录、后台、任务队列
    

  

## **11.4 可维护性**

- JSON Schema 统一
    
- 导出与展示基于同一结果源
    
- 配置和模板边界清晰
    
- 模块职责单一
    

  

## **11.5 可观测性**

- 有清晰日志
    
- 有阶段状态
    
- 可查看趋势模板更新时间与摘要
    

  

## **11.6 成本约束**

- 一期不直接做成品视频生成
    
- 一期不引入重型队列
    
- 一期不引入实时趋势分析
    
- 一期不引入复杂历史管理

# **12. 对开发 Agent 的明确约束**

  

## **12.1 必须遵守**

- 一期核心产物必须是**结构化结果包**
    
- 不得擅自做成视频生成一体机
    
- 不得擅自引入账号体系和历史记录
    
- 不得擅自引入复杂后台系统
    
- 不得将 Markdown 当作内部主数据
    
- 不得绕过外部设计引擎
    

  

## **12.2 不得改动的架构原则**

- 前端 Next.js
    
- 后端 FastAPI
    
- 前后端分离
    
- PostgreSQL 存模板与配置，不存用户历史结果
    
- JSON Schema 是唯一源数据结构
    
- Markdown / JSON / payload 全部从同一源结构派生
    

  

## **12.3 可接受的简化**

- 一期可先接一个真实模型，但必须留模型适配层
    
- 趋势分析器可先基于有限样本
    
- 页面样式可简洁
    
- 结果质量可先以结构完整和逻辑合理为优先
    

  

## **12.4 不可接受的偷懒实现**

- 直接输出一段大文本冒充结果包
    
- 平台适配层只写空泛文案
    
- 趋势能力无模板来源
    
- Video payload 没有结构，只是字符串拼接
    
- 前端页面硬编码结果内容
    
- Markdown 反向成为业务真值
    

  

## **12.5 验收标准**

  

至少满足：

- 用户可完成五步输入
    
- 生成页可展示五个阶段状态
    
- 系统可输出五层结果包
    
- 结果页可展示总览、脚本、平台适配
    
- 中间分析可见
    
- Markdown 导出成功
    
- JSON 导出成功
    
- Video payload 接口返回结构化数据
    
- 趋势模板可通过命令行/API 更新
    
- 模板读取失败有回退逻辑
