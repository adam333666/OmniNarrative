# M96 N6 移动端密度与阅读节奏优化

## 1. 背景

`N5` 已把四页主链的核心文案切到用户任务口吻。

`N6` 继续在现有视觉方向下优化：

- 移动端信息层级
- 长内容阅读负担
- 页面节奏与 CTA 落点

## 2. 本轮原则

- 不重写现有页面结构
- 不新增大块自写重特效
- 优先复用已在页面中的上游展示材料
- 主要通过响应式布局、间距、顺序和卡片密度优化体验

## 3. 本轮重点

### 3.1 首页

- Hero 在小屏下改成更直接的一列阅读顺序
- CTA、统计卡和 feature 区在小屏下减少横向压力

### 3.2 创建页

- 输入工作台在小屏下避免双栏挤压
- 趋势侧栏在移动端回落成普通顺序块
- 底部动作区更容易点击

### 3.3 结果页

- 小屏下减少长列表的横向并列
- 压缩卡片内边距，降低长内容滚动疲劳
- 保持“先看结果、再看依据、再导出”的阅读节奏

## 4. 完成判定

满足以下条件即可视为 `N6` 当前范围完成：

1. 首页、创建页、结果页在移动端的关键信息层级更清楚
2. 未新增大块自写重特效
3. 仍建立在现有 `react-bits-main` 与既有展示组件上
