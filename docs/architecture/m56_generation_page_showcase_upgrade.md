# M56 生成过渡页展示升级

## 1. 目标

把当前“能轮询状态”的生成过渡页升级为更适合展示与答辩的工作流舞台。

本轮重点不是改后端生成逻辑，而是让用户在生成等待阶段也能清楚感知：

- 系统当前处于哪个阶段
- 每个阶段在做什么
- 当前看到的状态来自真实轮询，而不是本地假进度

---

## 2. 本轮原则

- 优先提升演示效果与产品感知
- 继续复用已经接入的上游前端组件
- 不改变现有轮询与跳转逻辑

---

## 3. 本轮新增展示能力

### 3.1 生成页 Hero 舞台

生成页新增深色 Hero 舞台，展示：

- generation id
- 当前阶段
- 累计耗时
- 动态引导文案

### 3.2 阶段轨道

将原本简单的阶段列表升级成可解释的阶段轨道，明确标出：

- 已完成
- 进行中
- 待进入

并补上每个阶段的简短说明。

### 3.3 真实状态卡

在生成页独立强调：

- 当前 `status`
- 当前阶段耗时
- 当前轮询到的真实 `stage_message`

让演示时更容易说明“这不是静态动画，而是来自系统真实状态”。

---

## 4. 直接复用的上游组件

本轮继续复用已从 `react-bits-main` 接入的：

- `TextType`
- `DotGrid`

---

## 5. 验收标准

- 生成页保留原有轮询与完成后自动跳转逻辑
- 页面展示升级为更完整的工作流舞台
- 前端生产构建通过
