# M20 导出动作面板增强

## 目标

在 `M19` 完成分析面板摘要卡增强之后，继续整理结果页左侧栏的“导出动作”区域，把它从简单按钮列表升级成更清晰的操作面板。

本阶段目标：
- 将 JSON、Markdown、Video Payload 三个导出动作重组为更清晰的动作卡。
- 保留现有导出 URL 真值来源，不新增后端接口。
- 让导出区域与左侧总览卡、分析卡形成更统一的工作台式阅读与操作节奏。

## 原材料来源

本阶段直接参考：
- `/home/admin2/smy/upstream-materials/react-bits-main/src/ts-default/Components/CardNav/CardNav.tsx`

## 适配原则

- 保留上游 `CardNav` 的核心语义：通过分组动作卡强化导航目标与操作入口的辨识度。
- 不引入 `gsap`、汉堡菜单、展开动画等上游重交互能力，只借“动作卡 + CTA 链接”的组织方式。
- 导出动作区只重组现有 `getJsonExportUrl`、`getMarkdownExportUrl`、`getVideoPayloadUrl` 的前端调用，不创造新的导出通道。
- 保持导出动作语义清晰，让用户一眼区分“结构化导出”“阅读导出”“下游执行导出”。

## 变更范围

### 左侧导出动作区
- 将原有三个按钮升级为三张动作卡。
- 每张动作卡包含动作标签、用途说明与独立 CTA。
- 保留原有导出能力，不改目标链接行为。

### 结果页样式
- 新增导出动作面板组件与样式文件。
- 允许对结果页局部样式做最小调整，以适配新的导出动作区结构。

## 当前阶段明确不做

- 不改动后端 schema 与接口。
- 不新增导出格式。
- 不引入新的第三方依赖。

## 验证要求

- 变更后执行 `npm run build`。
- 同步更新 `docs/changelog.md`。
- 以同一批次 Git 提交收口。
