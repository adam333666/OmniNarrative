# M19 分析面板摘要卡增强

## 目标

在 `M18` 完成总览层摘要卡增强之后，继续整理结果页左侧栏里仍然偏线性的“分析面板”，把它升级成更利于快速扫描的摘要结构。

本阶段目标：
- 将趋势摘要、受众信息、兴趣标签重组为更清晰的卡片化摘要区。
- 保留现有 `analysis` 真值字段，不新增后端接口或派生字段。
- 让左侧分析面板与总览层、平台层、机器层、多模态层保持统一的摘要阅读节奏。

## 原材料来源

本阶段直接参考：
- `/home/admin2/smy/upstream-materials/react-bits-main/src/ts-default/Components/ProfileCard/ProfileCard.tsx`

## 适配原则

- 保留上游 `ProfileCard` 的核心语义：通过更清晰的信息分区提升摘要信息的辨识度。
- 不引入倾斜交互、头像卡语义或额外视觉噪音，只借“信息分层卡片”的组织方式。
- 分析面板只重组现有 `analysis.trend_summary` 与 `analysis.audience_profile` 字段，不创造新的业务真值。
- `AnimatedDecisionList` 继续保留为分析结论区，避免和摘要卡混合在一起造成阅读漂移。

## 变更范围

### 左侧分析面板
- 将趋势摘要升级为独立的高优先级摘要卡。
- 将受众原文与年龄猜测改为并列摘要卡。
- 将兴趣标签保留为独立标签区，继续承接现有真值。
- 将设计决策区继续置于摘要区之后，维持“摘要 -> 标签 -> 决策”的阅读顺序。

### 结果页样式
- 新增分析面板摘要组件与样式文件。
- 允许对结果页局部样式做最小调整，以适配新的信息分区。

## 当前阶段明确不做

- 不改动后端 schema 与接口。
- 不新增趋势评分、受众评分等前端自造字段。
- 不引入新的第三方依赖。

## 验证要求

- 变更后执行 `npm run build`。
- 同步更新 `docs/changelog.md`。
- 以同一批次 Git 提交收口。
