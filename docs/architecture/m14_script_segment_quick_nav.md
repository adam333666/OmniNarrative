# M14 脚本层段落快速导航增强

## 目标

在 `M13` 完成结果页分区 Pill 导航增强之后，继续从本地 `react-bits-main` 源码中借成熟的卡片入口语义，提升脚本层长内容的定位效率。

本阶段目标：
- 为脚本层新增一个“段落快速导航”，让用户可以直接跳到具体脚本段落。
- 继续保持脚本数据结构、结果结构和导出契约不变。
- 不引入 `gsap`、`react-icons` 或其他新依赖。

## 原材料来源

本阶段直接参考：
- `/home/admin2/smy/upstream-materials/react-bits-main/src/ts-default/Components/CardNav/CardNav.tsx`

## 适配原则

- 保留上游 `CardNav` 的核心语义：用一组结构清晰的入口卡片承接长内容中的高价值跳转点。
- 不复用上游的菜单展开动画、图标依赖和复杂布局，只借“卡片式导航入口”的信息结构。
- 使用原生锚点和轻量 CSS，实现段落定位与快速扫描。
- 样式继续服从当前暖色结果页视觉，不引入新的视觉体系。

## 变更范围

### 脚本层
- 在脚本层内容开始前新增一组段落快速导航卡。
- 每个导航项至少包含：段落编号、段落标题、跳转入口。
- 每个脚本段落卡增加稳定锚点 id。

### 结果页样式
- 新增脚本段落导航容器与卡片样式。
- 保持现有段落内容、平台层、多模态层和机器层结构不变。

## 当前阶段明确不做

- 不引入 `gsap`。
- 不引入图标库。
- 不改动后端结果结构与接口。

## 验证要求

- 变更后执行 `npm run build`。
- 同步更新 `docs/changelog.md`。
- 以同一批次 Git 提交收口。
