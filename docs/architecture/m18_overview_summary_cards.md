# M18 总览层摘要卡增强

## 目标

在 `M17` 完成平台层摘要卡增强之后，继续统一结果页顶部的阅读节奏，让总览层从“标题 + 一段说明 + 少量 MetaRow”升级为更清晰的摘要卡结构。

本阶段目标：
- 将总览层关键信息重组为更适合快速理解的摘要卡。
- 保留现有字段真值，不新增后端派生接口。
- 让总览层与平台层、机器层、多模态层形成更一致的摘要阅读模式。

## 原材料来源

本阶段直接参考：
- `/home/admin2/smy/upstream-materials/react-bits-main/src/ts-default/Components/ProfileCard/ProfileCard.tsx`

## 适配原则

- 保留上游 `ProfileCard` 的核心语义：通过更清晰的信息分区强化关键身份信息/摘要信息的可识别性。
- 不引入倾斜交互和视觉噪音，只借“结构化摘要卡”的语义。
- 总览层摘要卡只重组现有 `overview` 字段，不创造新的业务字段。
- 总览层的大标题和一句话梗概仍然保留为首屏焦点，不被摘要卡替代。

## 变更范围

### 总览层
- 将主标题、一句话梗概之外的关键信息改为摘要卡展示。
- 优先承载：平台目标、内容定位、风格摘要等已有真值字段。

### 结果页样式
- 新增总览层摘要卡容器与卡片样式。
- 不改动脚本层、平台层、多模态层、机器层当前结构。

## 当前阶段明确不做

- 不改动后端 schema 与接口。
- 不调整总览层字段定义。
- 不新增复杂交互。

## 验证要求

- 变更后执行 `npm run build`。
- 同步更新 `docs/changelog.md`。
- 以同一批次 Git 提交收口。
