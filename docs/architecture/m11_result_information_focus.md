# M11 结果页信息聚焦列表增强

## 目标

在 `M9` 完成结果页基础交互增强、`M10` 完成首页流程与指标增强之后，继续从本地 `react-bits-main` 源码中借成熟的信息型组件语义，提升结果页分析信息的可读性。

本阶段目标：
- 让“关键设计决策”从普通列表升级为可聚焦、可快速扫读的信息列表。
- 继续保持结果页的数据契约、导出契约和结果真值完全不变。
- 只做展示层增强，不引入新的运行时依赖。

## 原材料来源

本阶段直接参考：
- `/home/admin2/smy/upstream-materials/react-bits-main/src/ts-default/Components/AnimatedList/AnimatedList.tsx`

## 适配原则

- 保留上游 `AnimatedList` 的核心语义：列表项在进入视口后有轻量入场感，当前聚焦项有更明显的视觉强调。
- 不引入 `motion/react`；使用 CSS transition、轻量状态管理和现有 React 生命周期实现最小等价交互。
- 保留键盘可达性与鼠标悬停聚焦能力，但不复制原组件的全部滚动阴影和动画细节，避免为了效果扩张复杂度。

## 变更范围

### 结果页分析面板
- 将“关键设计决策”区改为信息聚焦列表。
- 默认高亮第一项，支持 hover / focus 切换当前聚焦项。
- 当前聚焦项在视觉上需要明显区别于其他条目。

### 结果页样式
- 为结果页新增一组列表容器、聚焦态和描述态样式。
- 现有 `interest_tags`、`avoid_patterns`、Markdown 预览、导出按钮都保持原逻辑。

## 当前阶段明确不做

- 不改动后端接口与结果结构。
- 不引入 `motion/react`。
- 不对脚本层、多模态层或机器层做结构调整。

## 验证要求

- 变更后执行 `npm run build`。
- 同步更新 `docs/changelog.md`。
- 以同一批次 Git 提交收口。
