# M13 结果页分区 Pill 导航增强

## 目标

在 `M11` 完成结果页信息聚焦列表增强、`M12` 完成标题层级增强之后，继续从本地 `react-bits-main` 源码中借成熟的导航语义，提升结果页长页面的浏览效率。

本阶段目标：
- 为结果页增加一个轻量分区导航，让用户能快速跳到总览、Markdown、脚本、平台、多模态、机器层。
- 继续保持结果结构、导出结构与后端接口完全不变。
- 不引入 `gsap`、`react-router-dom` 或其他新依赖。

## 原材料来源

本阶段直接参考：
- `/home/admin2/smy/upstream-materials/react-bits-main/src/ts-default/Components/PillNav/PillNav.tsx`

## 适配原则

- 保留上游 `PillNav` 的核心语义：以圆角 pill 项表达当前页面中多个高价值分区入口。
- 不复用上游的 `gsap` 动画和路由依赖，只借它的导航结构与视觉语义。
- 使用原生锚点和轻量 CSS 实现最小等价浏览体验。
- 配色与层次继续服从当前暖色主调，不复制上游深色导航样式。

## 变更范围

### 结果页内容区
- 新增一个分区导航条，提供以下跳转入口：总览层、Markdown、脚本层、平台层、多模态层、机器层。
- 为对应内容区增加稳定锚点 id。

### 结果页样式
- 新增 pill 导航容器、按钮和 sticky 表现样式。
- 不改动分析面板、导出按钮与数据展示内容本身。

## 当前阶段明确不做

- 不引入 `gsap`。
- 不引入客户端滚动监听高亮状态机。
- 不改动任何 API 与数据结构。

## 验证要求

- 变更后执行 `npm run build`。
- 同步更新 `docs/changelog.md`。
- 以同一批次 Git 提交收口。
