# M17 平台层摘要卡增强

## 目标

在 `M16` 完成多模态层摘要卡增强之后，继续提升结果页平台层的扫描效率，把目前偏线性的字段堆叠改成更适合浏览和比较的摘要卡结构。

本阶段目标：
- 将平台层关键信息从 `MetaRow` 线性列表重组为摘要卡网格。
- 保留原有字段真值，不新增后端派生接口。
- 让平台层与当前结果页其他摘要区形成更一致的阅读层次。

## 原材料来源

本阶段直接参考：
- `/home/admin2/smy/upstream-materials/react-bits-main/src/ts-default/Components/ProfileCard/ProfileCard.tsx`

## 适配原则

- 保留上游 `ProfileCard` 的核心语义：以更强的卡片边界和信息分区呈现高价值内容块。
- 不引入倾斜交互、头像、粒子或额外依赖，只借“信息身份卡”的结构语义。
- 平台层摘要卡只重组现有字段，不创造新的业务字段。
- `avoid_patterns` 继续保留为独立风险提示区，不与主摘要卡混写。

## 变更范围

### 平台层
- 将平台策略、受众适配、钩子设计原因、节奏结构原因、发布文案建议改为摘要卡网格。
- 保持 `avoid_patterns` 为单独区域。

### 结果页样式
- 新增平台层摘要卡容器、标题、正文样式。
- 不改动脚本层、多模态层、机器层的当前结构。

## 当前阶段明确不做

- 不改动后端 schema 与接口。
- 不引入交互倾斜或复杂动效。
- 不隐藏任何现有平台层字段。

## 验证要求

- 变更后执行 `npm run build`。
- 同步更新 `docs/changelog.md`。
- 以同一批次 Git 提交收口。
