# M59 首页品牌化展示升级

## 1. 背景

当前系统的输入页、生成页、结果页已经具备较强的演示感，但首页仍然相对偏弱，无法充分承担“品牌入口 + 能力总览 + 引导进入”的职责。

根据当前总计划，这一轮应进入：

- `P2` 首页完整品牌化展示

## 2. 本轮目标

把首页从“已有基础品牌入口”升级成真正适合演示和答辩的产品起点：

- 更清楚地展示四个核心功能
- 让首页视觉强度接近后续三页
- 继续严格复用上游成熟前端组件，而不是重新自造一套动画卡片系统

## 3. 具体实现

### 3.1 复制上游 `CardSwap`

本轮直接从 `react-bits-main` 复制并适配：

- `CardSwap.tsx`
- `CardSwap.css`

落地为：

- `frontend/components/upstream/card-swap.tsx`
- `frontend/components/upstream/card-swap.module.css`

用途：

- 在首页 Hero 区右侧做“四个核心功能”翻卡展示

### 3.2 首页 Hero 重构

`creative-hero` 改为双栏 Hero：

- 左侧：品牌主标题、typed selling line、CTA
- 右侧：核心能力翻卡组

并继续复用已接入的：

- `DotGrid`
- `TextType`
- `ShinyText`

### 3.3 更显式地表达四个核心卖点

翻卡内容直接围绕这四件事展开：

- 内部设计一致性
- 外部趋势传播增强
- 超重型结构化成品
- 简单上手主流程

这样首页不再只是“欢迎页”，而是直接表达系统价值。

## 4. 验证

- `cd frontend && npm run build`

## 5. 阶段意义

这轮对应总计划中的：

- `P2` 首页完整品牌化展示

本轮完成后，首页开始接近与输入页、生成页、结果页同等级的展示起点，有助于把整条演示链的第一印象拉起来。
