# M22 Markdown 预览大纲增强

## 目标

在 `M21` 完成左侧结果总览卡摘要增强之后，继续提升主内容区 `Markdown` 预览的阅读效率，把它从“标题 + 正文展示”升级成“预览说明 + 大纲入口 + 正文预览”的结构。

本阶段目标：
- 为 Markdown 预览增加一层轻量大纲入口，帮助用户快速跳转到正文关键段落。
- 保留现有后端 `/export/md` 真值输出，不新增后端接口或第二套正文内容。
- 让 Markdown 区在阅读方式上更接近结果页其余区域的“先摘要、后正文”节奏。

## 原材料来源

本阶段直接参考：
- `/home/admin2/smy/upstream-materials/react-bits-main/src/ts-default/Components/PillNav/PillNav.tsx`

## 适配原则

- 保留上游 `PillNav` 的核心语义：通过轻量锚点入口提升长内容浏览效率。
- 不引入 `gsap`、路由联动或复杂 hover 动画，只借“轻量导航 pill”的组织方式。
- Markdown 正文仍然直接渲染后端 `/export/md` 输出，不额外生成第二套正文。
- 大纲信息只从当前 Markdown 文本中提取标题结构，不创造新的业务真值。

## 变更范围

### Markdown 预览区
- 增加预览说明卡，明确当前预览来源与阅读模式。
- 增加从 Markdown 标题提取的大纲入口区。
- 为正文标题加上稳定锚点，支持就地跳转。

### 结果页样式
- 调整 Markdown 预览区样式，适配新的三段式结构。
- 不改动脚本层、平台层、多模态层、机器层当前结构。

## 当前阶段明确不做

- 不改动后端 schema 与接口。
- 不新增 Markdown 编辑能力。
- 不引入新的第三方依赖。

## 验证要求

- 变更后执行 `npm run build`。
- 同步更新 `docs/changelog.md`。
- 以同一批次 Git 提交收口。
