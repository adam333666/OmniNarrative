# M42 前端错误恢复与重试交互补强

## 1. 文档定位

本文档用于补齐当前前端仍较薄弱的一项产品缺口：

> 为生成状态页与结果页增加更明确的错误恢复与重试交互，
> 让用户在轮询失败、生成失败、结果包加载失败时，
> 不再只能看到一条错误文案，而是拥有清晰的恢复路径。

本文档受以下事实源约束：
- `docs/PRD.md`
- `docs/stage_records/stage_02_project_handoff_summary_v2.md`
- `docs/stage_records/stage_03_next_phase_deepening_plan_v1.md`
- `docs/changelog.md`

---

## 2. 当前问题

当前前端在错误态上的主要问题是：
- `generation-status-client.tsx` 出错时只显示文案和“返回重新开始”
- `result-view-client.tsx` 出错时只显示一条错误文本

这会导致：
- 用户无法显式触发重试
- 用户缺少“返回状态页 / 回到创建页”的清晰动作
- 页面恢复路径不够产品化

---

## 3. 本轮目标

本轮目标是最小代价补强错误恢复能力：

1. 生成状态页支持手动重试轮询
2. 生成失败/超时时提供返回创建页的清晰动作
3. 结果页加载失败时提供“重试加载 / 返回状态页 / 重新开始”
4. 保持当前页面结构不做大改

---

## 4. 技术方案

### 4.1 生成状态页

- 增加手动“立即重试状态拉取”按钮
- 错误卡片中保留当前阶段提示
- 若状态已进入终态，避免继续自动轮询

### 4.2 结果页

- 增加显式“重新加载结果包”按钮
- 提供返回生成状态页与重新开始两个恢复入口
- 在错误态保留 generation_id 便于排查
