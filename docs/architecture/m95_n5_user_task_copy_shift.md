# M95 N5 前端文案用户任务化改写

## 1. 背景

`N4/N7` 之后，系统已经具备稳定回归入口与评测材料。

当前前端的主要差距不再是“能不能展示”，而是：

- 仍有一部分文案站在系统视角解释自己
- 用户更关心的是：
  - 这一步要做什么
  - 为什么要做这一步
  - 接下来会发生什么

## 2. 本轮原则

- 不改变现有展示层级
- 不推翻现有视觉方向
- 只把核心文案从展示口吻切到用户任务口吻
- 继续复用现有 `react-bits-main` 展示材料，不新造展示系统

## 3. 前后对照清单

### 3.1 首页

- 改写前：
  - 更偏“这是一个展示优先的工作台”
- 改写后：
  - 更偏“你可以从一句想法直接开始，系统会帮你长成可用方案”

### 3.2 创建页

- 改写前：
  - 更偏“输入会被系统如何处理”
- 改写后：
  - 更偏“你这一步该填什么，填完会得到什么”

### 3.3 生成页

- 改写前：
  - 更偏“生成过程本身是可展示工作流”
- 改写后：
  - 更偏“你正在等待什么，什么时候会自动进入下一步”

### 3.4 结果页

- 改写前：
  - 更偏“这里展示了一个很强的结构化系统”
- 改写后：
  - 更偏“这是一份你现在就能带走、导出、继续改写的方案”

## 4. 完成判定

满足以下条件即可视为 `N5` 当前范围完成：

1. 首页、创建页、生成页、结果页四页都完成一轮核心文案改写
2. 页面展示层级保持不变
3. docs 中已保留一轮“改写前 / 改写后”的对照说明
