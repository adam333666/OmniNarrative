# M55 输入向导趋势预览与上游前端特效接入

## 1. 目标

把当前开发重心明确切回“展示优先 + 产品可演示”。

本轮同时完成两件事：

- 在输入向导页补一个真实可用的“平台趋势预览”功能
- 直接从 `/home/admin2/smy/upstream-materials/react-bits-main` 复制成熟前端组件源码接入现有页面

---

## 2. 本轮原则

- 前端展示和美观性优先级提升
- 但展示增强必须尽量建立在真实系统功能之上，而不是纯静态假面板
- 关键视觉效果优先整块复用上游成熟组件源码，而不是仓内重新手写

---

## 3. 本轮直接复用的上游代码

本轮直接从 `react-bits-main` 复制并接入：

- `src/ts-default/TextAnimations/TextType/TextType.tsx`
- `src/ts-default/TextAnimations/TextType/TextType.css`
- `src/ts-default/Backgrounds/DotGrid/DotGrid.tsx`
- `src/ts-default/Backgrounds/DotGrid/DotGrid.css`

当前项目在本地进行了最小必要适配：

- 改为 CSS Module 导入
- 改为当前目录与命名风格
- 保留原组件核心运行逻辑

---

## 4. 本轮新增功能

### 4.1 输入向导平台趋势预览

在输入向导页新增趋势预览侧栏，直接调用现有后端：

- `GET /api/v1/config/trend-templates`

用户在选择平台和内容类型时，可以即时看到：

- 当前平台主趋势摘要
- 当前热点摘要
- 其他同平台趋势条目

### 4.2 展示层特效接入

趋势预览侧栏直接接入上游组件：

- `TextType` 作为动态引导文案
- `DotGrid` 作为交互背景层

这样输入向导页不再只是表单，而是更像一个“系统已经开始理解平台和趋势”的工作台。

---

## 5. 本轮意义

这轮属于当前阶段更高优先级的工作，因为它同时满足三件事：

- 直接提升前端展示效果
- 直接增强产品可演示性
- 没有牺牲系统真实性，而是把后端已有趋势能力前置到用户主流程中

---

## 6. 验收标准

- 输入向导页出现趋势预览侧栏
- 预览内容来自后端真实趋势模板接口
- 页面直接使用从 `react-bits-main` 复制接入的上游组件
- 前端生产构建通过
