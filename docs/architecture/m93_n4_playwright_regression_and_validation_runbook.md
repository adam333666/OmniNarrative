# M93 N4 Playwright 回归与统一验证顺序

## 1. 背景

`N4` 的目标不是新造一套测试系统，而是把已经进入仓库的成熟验证材料真正接上主回归入口。

当前仓库已经有：

- `Playwright`
- `demo_regression.sh`
- `backend_test.sh`
- `README`
- `v1.0` 评测交接文档

但还缺一层收口：

- `Playwright` 尚未成为统一回归顺序的一部分
- README 与评测文档中的验证顺序还没有完全对齐

## 2. 本轮目标

本轮作为 `N4` 第一批闭环，锁定三件事：

1. 让 `Playwright` 真正进入统一回归主入口
2. 至少覆盖：
   - 1 条完整主流程
   - 1 条内部工具链 / 导出链
3. 把 README 与评测文档统一到同一套验证顺序

## 3. 验证顺序真值

统一建议顺序为：

1. `./scripts/backend_test.sh <关键后端回归集合>`
2. `cd frontend && npm run build`
3. `cd frontend && npm run test:e2e`

其中：

- 第 1 步验证后端关键链路与结构化真值
- 第 2 步验证前端生产构建与类型收口
- 第 3 步验证页面级闭环与内部工具链

## 4. Playwright 覆盖要求

### 4.1 主流程

至少覆盖：

- 创建页
- 生成页
- 结果页

### 4.2 内部工具链 / 导出链

至少覆盖其一，并优先都覆盖：

- checkpoint 控制台
- 趋势控制台
- 结果页导出接口

## 5. 完成判定

满足以下条件即可视为 `N4` 第一批完成：

1. `Playwright` 已进入统一回归主入口
2. `Playwright` 至少覆盖 1 条完整主流程和 1 条内部工具链 / 导出链
3. README 与评测交接文档对验证顺序的表述一致
