# M78 趋势来源轨迹真值链

## 1. 任务定位

本轮对应里程碑：

- `M-P1-A 趋势入口主路径成熟化`
- `M-P1-C 趋势增强演示闭环`

目标不是继续补一个展示卡片，而是把“趋势从哪来”做成一条可持久化、可接口透传、可前端展示的真值链。

## 2. 变更目的

在此前实现中，系统已经具备：

- `RSSHub -> Crawl4AI -> Instructor` 的趋势采集与结构化归纳主链
- 创建页趋势策略展示
- 结果页趋势影响展示
- 内部趋势控制台

但仍然缺少一个关键环节：

- 当前趋势摘要虽然存在，却没有把“使用了哪些来源条目、入口来自哪里”沉淀成真值

这会带来两个问题：

- 演示时只能说“系统参考了趋势”，却不能直接展示“参考了什么”
- 内部排查时只能看到摘要和热点，缺少来源轨迹

## 3. 本轮实现

### 3.1 后端真值层

- 在 `PlatformTrendTemplate` 和 `TrendTemplateSummary` 中新增 `source_trace`
- 引入 `TrendSourceTraceItem`
- 数据库 `platform_trend_templates` 新增 `source_trace` JSON 列
- Alembic 新增迁移 `20260327_182500_add_trend_source_trace.py`

### 3.2 趋势采集层

- `RSSHub` 路径下，把 feed item 的 `title / link / description` 转成来源轨迹
- `Crawl4AI` 路径下，把抓取文档标题、URL 和摘录转成来源轨迹
- 结构化归纳成功时保留来源轨迹
- 结构化归纳失败走 excerpt fallback 时也保留来源轨迹

### 3.3 接口与消费层

- `/config/trend-templates` 透传 `source_trace`
- `/internal/trend-summary/{platform}` 透传 `source_trace`

### 3.4 前端展示层

- 创建页趋势预览面板新增来源轨迹区块
- 内部趋势控制台新增来源轨迹列表

## 4. 价值判断

这轮完成后，趋势增强链不再只有“入口存在”和“摘要存在”，而是补上了：

- 来源可追溯
- 演示可点验
- 排查可落到具体条目

它直接增强了两个里程碑：

- `M-P1-A`：主路径不仅存在，而且有来源真值可证明
- `M-P1-C`：演示闭环里“趋势入口存在”与“趋势被结构化归纳”都更容易被讲清

## 5. 后续衔接

下一步可以继续在这条链上做两类深化：

1. 首页或内部控制台进一步强调“当前样例平台使用了哪些真实入口”
2. 结果页把趋势影响与来源轨迹关联得更近，形成更完整的增强闭环
