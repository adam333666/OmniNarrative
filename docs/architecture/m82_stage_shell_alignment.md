# M82 主流程页头展示强度对齐

## 1. 对应里程碑

- `M-P2-B 四页主链展示强度一致`

## 2. 背景

在此前几轮增强后：

- 首页已经具备较强的品牌入口感
- 创建页、生成页、结果页的主体内容也都能演示

但它们之间仍有一个明显差异：

- 三张内页的页头层仍偏普通文档壳
- 首页的舞台感明显更强

这会让四页主链在展示强度上出现落差。

## 3. 本轮实现

- 参考 `react-bits-main` 的 `GlassSurface` 结构语义新增项目内适配版
- 新增共享的 `FlowStageBanner`
- 创建页、生成页、结果页统一接入阶段英雄条
- 阶段英雄条统一展示：
  - 当前页面在主流程里的角色
  - 这页负责解决什么
  - 下一步去哪里
  - 当前页的核心亮点
  - 当前页的快捷动作

## 4. 结果判断

这轮完成后：

- 首页有完整品牌入口与结果示例
- 创建页有阶段英雄条和输入工作台
- 生成页有阶段英雄条和生成舞台
- 结果页有阶段英雄条和结果工作台

因此四页主链已经都达到“可独立演示”的展示强度，`M-P2-B` 可以从未完成推进到已完成。
