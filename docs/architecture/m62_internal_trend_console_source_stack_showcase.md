# M62 内部趋势控制台来源栈展示

## 1. 背景

上一轮已经补了轻量内部趋势控制台，但页面仍然主要停留在“有数据可看”。为了更清楚说明系统如何复用成熟仓库完成趋势增强链，还需要把来源栈本身展示出来。

## 2. 本轮目标

在不新增一套手写展示组件的前提下，直接复用项目里已经接入的上游 `CardSwap`，把趋势来源栈可视化：

- `RSSHub`
- `Crawl4AI`
- `Instructor`

## 3. 具体实现

本轮没有新增新的自写动画组件，而是：

- 继续复用已经从 `react-bits-main` 复制进项目的 `CardSwap`
- 在 `/internal/trends` 页面新增来源栈翻卡区
- 用这组翻卡解释外部趋势增强链的三层结构

## 4. 验证

- `cd frontend && npm run build`

## 5. 阶段意义

这轮仍对应总计划中的：

- `P3` 轻量内部管理能力

但重点是让这条链不仅“能用”，还“能被讲清楚”，同时继续保持强胶水原则，不额外造轮子。
