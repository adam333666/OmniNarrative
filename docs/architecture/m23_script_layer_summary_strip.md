# M23 脚本层摘要条带增强

## 目标

在 `M22` 完成 Markdown 预览大纲增强之后，继续提升结果页脚本层的扫读效率，把它从“导航 + 完整段落详情”升级成“导航 + 摘要条带 + 完整段落详情”的结构。

本阶段目标：
- 为每个脚本段落增加一层轻量摘要入口，帮助用户先扫读目标与节奏，再进入完整字段。
- 保留现有脚本层真值字段，不新增后端接口或第二套脚本内容。
- 让脚本层阅读方式更接近结果页其余区域的“先摘要、后详情”节奏。

## 原材料来源

本阶段直接参考：
- `/home/admin2/smy/upstream-materials/react-bits-main/src/ts-default/Components/CardNav/CardNav.tsx`

## 适配原则

- 保留上游 `CardNav` 的核心语义：通过轻量卡片入口提升多段内容的浏览效率。
- 不引入展开动画或额外交互依赖，只借“摘要卡入口”的组织方式。
- 脚本摘要条带只重组现有 `segment_goal`、`emotion`、`rhythm`、`segment_title` 字段，不创造新的业务真值。
- 完整段落详情仍保留原有结构，不被摘要条带替代。

## 变更范围

### 脚本层
- 在段落快速导航之后增加摘要条带。
- 每张摘要卡包含段落编号、标题、目标摘要和情绪/节奏信息。
- 保留原有完整段落详情区与锚点跳转行为。

### 结果页样式
- 新增脚本摘要条带组件与样式文件。
- 不改动平台层、多模态层、机器层当前结构。

## 当前阶段明确不做

- 不改动后端 schema 与接口。
- 不新增脚本编辑能力。
- 不引入新的第三方依赖。

## 验证要求

- 变更后执行 `npm run build`。
- 同步更新 `docs/changelog.md`。
- 以同一批次 Git 提交收口。
