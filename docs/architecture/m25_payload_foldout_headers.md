# M25 多模态层与机器层折叠头增强

## 目标

在 `M24` 完成平台层风险提示面板增强之后，继续统一结果页底部折叠区的阅读节奏，把多模态层与机器层从默认 `details/summary` 头部升级成更明确的摘要头结构。

本阶段目标：
- 为多模态层与机器层增加统一的折叠头摘要。
- 保留现有 payload 摘要卡与原始 JSON 真值，不新增后端接口。
- 让两个折叠区形成“摘要头 + 摘要卡 + 原始 JSON”的完整节奏。

## 原材料来源

本阶段直接参考：
- `/home/admin2/smy/upstream-materials/react-bits-main/src/ts-default/Components/Stepper/Stepper.tsx`

## 适配原则

- 保留上游 `Stepper` 的核心语义：通过阶段头与当前状态提示强化“进入下一层内容前的认知准备”。
- 不引入动画、步骤切换或额外交互依赖，只借“阶段头 + 状态提示”的信息组织方式。
- 折叠头只重组现有 payload 的顶层规模信息，不创造新的业务真值。
- 摘要卡与原始 JSON 保持现有内容结构不变，折叠头只负责进入前的说明与导航。

## 变更范围

### 多模态层与机器层
- 新增统一的折叠头组件。
- 为两个折叠区展示层名称、用途说明、顶层字段规模和展开提示。
- 保留现有摘要卡与原始 JSON 展示区。

### 结果页样式
- 新增折叠头组件与样式文件。
- 允许对 `details/summary` 局部样式做最小调整，以适配新的摘要头结构。

## 当前阶段明确不做

- 不改动后端 schema 与接口。
- 不新增 payload 编辑能力。
- 不引入新的第三方依赖。

## 验证要求

- 变更后执行 `npm run build`。
- 同步更新 `docs/changelog.md`。
- 以同一批次 Git 提交收口。
