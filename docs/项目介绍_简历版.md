# multi-media 项目介绍（简历可用版）

## 1. 项目定位（一句话）
这是一个面向内容创作者的多模态叙事方案生成系统：用户从一句创意出发，经过结构化引导，获得可展示、可解释、可导出、可被下游视频生产链消费的执行级结果包。

## 2. 项目背景与价值
传统内容生成工具常停留在“单点产出”（只产文案、只产脚本或只产提示词），无法支撑从创意到执行的完整链路。本项目解决的是“创意落地断层”问题：
- 把模糊创意转成结构化创作输入
- 把平台差异、受众偏好、趋势信号显式纳入生成过程
- 把结果升级为可交付给后续制作流程的执行级数据包

## 3. 系统目标与范围（MVP）
按 PRD 定义，MVP 聚焦“展示优先 + 前后端分离 + 结构化输出闭环”：
- 五步向导式输入
- 分阶段生成与状态可视化
- 五层结果包输出
- 结果与导出同源（JSON / Markdown / Video Payload）
- 趋势链可触发、可展示、可追溯

明确不在一期范围：成品视频直接生成、账号体系、历史项目管理、生成后二次编辑、多用户生产级并发优化。

## 4. 技术架构（真实落地）
### 前端
- Next.js 应用路由，主流程为首页、创建页、生成页、结果页
- 通过统一 API 客户端完成请求、轮询、导出、错误透传
- 结果页承载多层结构展示、导出动作和内部追踪入口

### 后端
- FastAPI 提供主链路与内部接口
- 生成主链路：输入归一化 -> 画像解析 -> 趋势适配 -> 叙事生成 -> 结果组装
- 编排采用 LangGraph 状态图，并接入 checkpoint 持久化与恢复
- 结果真值入库，result/export/video-payload 同源派生

### 数据与工程化
- 数据库真值层承接任务状态、结果、事件轨迹
- Alembic 迁移治理
- Playwright + pytest 回归入口，支持连续评测
- Docker 化部署边界在 PRD 与阶段文档中已预留

## 5. 开发逻辑与流程（可写进简历的 10 步主线）
1. 先定义产品真值：完成 PRD，锁定用户场景、MVP 边界和结果包结构。
2. 再定义接口真值：先写输入/状态/结果/导出契约，保证前后端并行开发时不漂移。
3. 搭建前端主链路：先打通创建-生成-结果最短可用链路，再逐步增强展示密度和可读性。
4. 落地后端最小闭环：先让 generate/status/result 可跑通，再补强质量。
5. 做后端忠实度升级：分阶段完成数据库真值层、趋势采集链、模型网关、状态持久化。
6. 引入结构化输出网关：把模型响应约束到稳定 schema，减少格式漂移与消费端失败。
7. 升级编排可靠性：将执行链切到 LangGraph，接入 checkpoint 持久化、恢复与状态快照。
8. 加厚结果包：从“可读方案”升级为“执行级交付包”，补齐分镜帧、素材依赖、配音字幕对齐等字段。
9. 建立回归与评测体系：统一 demo_regression 和固定样例包，形成可复现验收入口。
10. 做产品化收口：文案用户任务化、移动端阅读节奏优化、内部控制台可解释性增强。

## 6. 你作为唯一作者可强调的个人贡献
- 独立完成从 PRD、API 契约、架构演进到前后端实现与测试交接的全流程。
- 以“文档先行 + 契约先行”方法控制复杂系统迭代，降低返工与跨层偏差。
- 将生成系统从演示原型推进为可持续评测的工程基线（可回归、可追踪、可恢复）。
- 在趋势增强、结构化输出、执行编排三条关键链路上完成实装与闭环。

## 7. 简历描述模板
### 7.1 技术导向版
- 独立设计并实现基于 FastAPI + Next.js 的多模态内容生成系统，构建从输入、生成、结果到导出的端到端闭环。
- 采用 LangGraph 重构生成编排，落地 checkpoint 持久化、失败归因与结果恢复，显著提升长链路任务稳定性与可排障性。
- 建立结构化输出与统一 schema 约束，将结果包扩展为执行级格式，支持 JSON/Markdown/Video Payload 同源导出。
- 打通趋势增强链（采集、归纳、注入、追溯），使外部信号可解释地参与内容设计与分发策略生成。

### 7.2 业务价值版
- 面向内容创作者打造零门槛创作工作台，将“模糊创意”转化为可直接推进制作的结构化方案。
- 通过五步向导与阶段化反馈降低首次使用门槛，提高从需求输入到结果获取的流程清晰度。
- 输出覆盖脚本、分镜、平台策略与执行建议的重型结果包，缩短创意到落地的协同路径。
- 建立固定样例与评测矩阵，使系统能力可重复验证、可对外演示、可持续迭代。

### 7.3 全栈叙事版
- 作为项目唯一作者，主导并完成产品定义、系统设计、前后端开发、测试与交接文档全链路建设。
- 前端实现创建-生成-结果主流程与多层结果展示，后端实现状态持久化、趋势注入、结果物化与导出服务。
- 通过契约驱动开发与阶段里程碑推进，在多轮迭代中保持接口稳定、页面可演示、结果可追溯。
- 构建统一回归入口与内部控制台，支持快速定位问题并保障版本连续可运行。

## 8. 面试可展开的技术亮点
- 为什么先做 API 契约再做大规模功能开发。
- 如何把结果真值统一，避免 result/export/video-payload 各自重算导致不一致。
- 为什么选择 LangGraph 以及 checkpoint 在真实故障恢复中的价值。
- 趋势增强链如何做到“进入主链”而不是“页面装饰”。
- 结构化输出网关如何降低模型响应不稳定带来的后续故障。
- 如何设计评测矩阵，让核心目标可观察、可比较、可复现。
